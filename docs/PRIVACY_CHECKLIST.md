# Health-data privacy checklist

- Publish a clear privacy policy and terms.
- Collect only fields used by a visible feature.
- Keep service keys server-side.
- Enable Supabase RLS on every client-owned table.
- Use private storage buckets and signed URLs.
- Add client export and deletion workflows.
- Record coach access and administrative changes.
- Define data retention periods.
- Encrypt backups and restrict production access.
- Add rate limits and abuse monitoring to API routes.
- Obtain legal review for the jurisdictions and client populations you serve.
- Avoid presenting projections as medical diagnosis or guaranteed outcomes.
